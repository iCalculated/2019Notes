Lecture

Floating Point Efficiency

FLOP refers to FLoatig point Operations Per Second, used to benchmark hardware.

Single flop may take three clock cycles, but a flop can start before the previous one ends (called pipelining). Enough flops stacked result in nearly 1 FLOP per cycle. Vectorized ops allow operation on multiple FPs in parallel. Thus, FP operations are referred to as superscalar.

3 of the world’s largest super computers live in the US. Number one has nearly 2.4 million cores and does 143 thousand TFlops/s. Number three (Sunway TaihuLight from China) has over 10 million cores.

Many Languages

Assembly is not a monolith, everyone has their own breed.

Notable architectures are:

MOS Technology 6502

8-bit operations, limited addressable memory, 1 general purpose register.

Used in notable gaming systems and early consumer electronics such as those made by Nintendo, Atari, and Commodore.

IBM Cell Microprocessor

Early 2000’s, large quantity of cores, large space, fast multimedia.

Absolute pain to program.

Used for Playstation 3 and Blue Gene.

Compiler

The chain of tools that translate from high level to lower level languages.

In the case of C, the process is

  1. Preprocessing (cpp)
  2. Compiler (gcc, g++)
  3. Assembler (as)
  4. Linker (ld)

This means we don’t need to write assembly by hand, which is a major plus. It also means that an Intel processor can generate assembly for other chips.

x86-64 Assembly

Intel’s child, targets Intel/AMC with 64-bit words.

Descended from IA32 for 32-bit systems, which descended from a 16-bit system itself. Everything is gloriously back-compatible, but also the language is full of cruft because of that.

Within the language alone there are multiple syntaxes. We will use the AT&T Syntax which made it’s way into GNU tools. Main differences from Intel’s own syntax is that % signs are included to indicate registers and there are operators to indicate operand size.

gcc -s file.c stops at assembly generation. Produces file.s (though file.asm is also used)

Oh no, assembly is a mess! Try using the -Og (optimize debugging) flag, it makes the assembly easier to work with.

Syntax

(some of this is AT&T specific)

# → comments
% → a register (variables). They have explicit names, the four original ones were ax, bx, cx and dx. Then it all went downhill.
$ → used for constants
mov → assignment
cmp → comparison
goto → oh no.

Conditionals are followed by an instruction that should be performed if not met.

Formatted with chunk label on first column, then instructions within indented. One instruction per line, followed by parameters.
